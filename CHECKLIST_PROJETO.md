# Checklist para Criação de Projeto

## 1. Planejamento Inicial

- [ ] Definir o tipo de aplicação (web, mobile, desktop, API, etc.)
- [ ] Listar funcionalidades principais do sistema
- [ ] Escolher tecnologia/framework principal
- [ ] Escolher linguagem de programação
- [ ] Definir banco de dados (se necessário)
- [ ] Identificar integrações externas (APIs, serviços)

## 2. Arquitetura do Sistema

- [ ] Projetar estrutura de pastas/diretórios
- [ ] Definir camadas (ex: frontend, backend, database)
- [ ] Definir responsabilidades de cada componente
- [ ] Desenhar comunicação entre módulos
- [ ] Definir contratos/interfaces entre partes
- [ ] Escolher padrão de arquitetura (MVC, Clean Architecture, etc.)

## 3. Configuração do Ambiente

- [ ] Inicializar projeto com template/boilerplate
- [ ] Configurar linter e formatter
- [ ] Configurar TypeScript (se aplicável)
- [ ] Configurar variáveis de ambiente
- [ ] Configurar scripts de build, dev, test
- [ ] Configurar banco de dados e migrations

## 4. Estrutura Base

- [ ] Criar estrutura de pastas por domínio/recurso
- [ ] Implementar sistema de rotas
- [ ] Implementar configuração global
- [ ] Implementar autenticação (se aplicável)
- [ ] Criar tipos/interfaces globais
- [ ] Configurar manejo de erros global

## 5. Desenvolvimento de Funcionalidades

- [ ] Desenvolver um módulo por vez
- [ ] Ao desenvolver, verificar integração com módulos existentes
- [ ] Manter consistência de nomenclatura
- [ ] Manter consistência de estilo de código
- [ ] Documentar decisões de implementação

## 6. Validação e Consistência

- [ ] Revisar se componente segue padrões estabelecidos
- [ ] Verificar se código segue DRY (Don't Repeat Yourself)
- [ ] Verificar se há acoplamento excessivo
- [ ] Garantir que mudanças não impactem outros módulos inesperadamente
- [ ] Executar testes unitários
- [ ] Executar testes de integração

## 7. Qualidade e Manutenção

- [ ] Adicionar README com instruções
- [ ] Documentar API (se aplicável)
- [ ] Configurar CI/CD (se aplicável)
- [ ] Configurar deploy automatizado (se aplicável)

## Boas Práticas de Interação com OpenCode

- ✅ Ser específico sobre funcionalidades e tecnologia
- ✅ Começar com escopo definido e iterar gradualmente
- ✅ Pedir validação de integração ao adicionar novos componentes
- ✅ Solicitar revisão de consistência periodicamente
- ✅ Definir contratos/interfaces antes de implementar módulos
- ✅ Perguntar sobre arquitetura antes de começar a codar

---

*Checklist baseado em práticas de desenvolvimento consistente e integrado.*
